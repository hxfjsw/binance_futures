## 三、详细表结构

### 3.1 users（用户表）

```sql
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    account         VARCHAR(100) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    name            VARCHAR(50),
    role            VARCHAR(20) NOT NULL DEFAULT 'user',
    bind_ip         VARCHAR(50),
    status          VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMP,
    last_login_ip   VARCHAR(50)
);

-- 索引
CREATE INDEX idx_users_account ON users(account);
CREATE INDEX idx_users_role ON users(role);
CREATE INDEX idx_users_status ON users(status);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 自增主键 |
| account | VARCHAR(100) | UK, NOT NULL | 登录邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希 |
| name | VARCHAR(50) | NULL | 员工姓名 |
| role | VARCHAR(20) | DEFAULT 'user' | admin / user |
| bind_ip | VARCHAR(50) | NULL | 绑定IP |
| status | VARCHAR(20) | DEFAULT 'active' | active / disabled / deleted |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |
| last_login_at | TIMESTAMP | NULL | 最后登录时间 |
| last_login_ip | VARCHAR(50) | NULL | 最后登录IP |

### 3.2 api_keys（API Key 账户表）

```sql
CREATE TABLE api_keys (
    id              SERIAL PRIMARY KEY,
    user_id         INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    label           VARCHAR(50) NOT NULL,
    api_key         VARCHAR(255) NOT NULL,
    api_secret      VARCHAR(255) NOT NULL,
    domain          VARCHAR(255) NOT NULL DEFAULT 'https://fapi.binance.com',
    margin_type     VARCHAR(20) NOT NULL DEFAULT 'cross',
    position_mode   VARCHAR(20) NOT NULL DEFAULT 'single',
    leverage        INT NOT NULL DEFAULT 5,
    is_active       BOOLEAN NOT NULL DEFAULT true,
    total_net_worth DECIMAL(36,18) DEFAULT 0,
    available_balance DECIMAL(36,18) DEFAULT 0,
    unrealized_pnl  DECIMAL(36,18) DEFAULT 0,
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_api_keys_user_id ON api_keys(user_id);
CREATE INDEX idx_api_keys_api_key ON api_keys(api_key);
CREATE INDEX idx_api_keys_active ON api_keys(is_active);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 自增主键 |
| user_id | INT | FK → users.id | 所属用户 |
| label | VARCHAR(50) | NOT NULL | 账户标签（账户1） |
| api_key | VARCHAR(255) | NOT NULL | 币安 API Key |
| api_secret | VARCHAR(255) | NOT NULL | 币安 API Secret（加密存储） |
| domain | VARCHAR(255) | DEFAULT 'https://fapi.binance.com' | 交易所域名 |
| margin_type | VARCHAR(20) | DEFAULT 'cross' | isolated/cross |
| position_mode | VARCHAR(20) | DEFAULT 'single' | single/dual |
| leverage | INT | DEFAULT 5 | 杠杆倍数 1-125 |
| is_active | BOOLEAN | DEFAULT true | 是否激活 |
| total_net_worth | DECIMAL(36,18) | DEFAULT 0 | 账户总净值（缓存） |
| available_balance | DECIMAL(36,18) | DEFAULT 0 | 可用余额（缓存） |
| unrealized_pnl | DECIMAL(36,18) | DEFAULT 0 | 未实现盈亏（缓存） |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

### 3.3 robot_configs（机器人配置表）

```sql
CREATE TABLE robot_configs (
    id              SERIAL PRIMARY KEY,
    user_id         INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    api_key_id      INT NOT NULL REFERENCES api_keys(id) ON DELETE CASCADE,
    robot_type      VARCHAR(50) NOT NULL,
    category        VARCHAR(20) NOT NULL,
    symbol          VARCHAR(20) NOT NULL,
    params          JSONB NOT NULL DEFAULT '{}',
    status          VARCHAR(20) NOT NULL DEFAULT 'stopped',
    start_mode      VARCHAR(20) NOT NULL DEFAULT 'immediate',
    price_mode      VARCHAR(20) NOT NULL DEFAULT 'normal',
    description     TEXT,
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    started_at      TIMESTAMP,
    stopped_at      TIMESTAMP
);

-- 索引
CREATE INDEX idx_robot_configs_user_id ON robot_configs(user_id);
CREATE INDEX idx_robot_configs_api_key_id ON robot_configs(api_key_id);
CREATE INDEX idx_robot_configs_type ON robot_configs(robot_type);
CREATE INDEX idx_robot_configs_category ON robot_configs(category);
CREATE INDEX idx_robot_configs_status ON robot_configs(status);
CREATE INDEX idx_robot_configs_symbol ON robot_configs(symbol);
CREATE INDEX idx_robot_configs_params ON robot_configs USING GIN(params);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 自增主键 |
| user_id | INT | FK | 所属用户 |
| api_key_id | INT | FK | 绑定的 API Key |
| robot_type | VARCHAR(50) | NOT NULL | 机器人类型标识 |
| category | VARCHAR(20) | NOT NULL | normal/strategy/ai/risk |
| symbol | VARCHAR(20) | NOT NULL | 交易对（TRXUSDT） |
| params | JSONB | DEFAULT '{}' | 策略参数（JSON） |
| status | VARCHAR(20) | DEFAULT 'stopped' | running/stopped/error |
| start_mode | VARCHAR(20) | DEFAULT 'immediate' | immediate/scheduled |
| price_mode | VARCHAR(20) | DEFAULT 'normal' | normal/safe |
| description | TEXT | NULL | 描述 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |
| started_at | TIMESTAMP | NULL | 启动时间 |
| stopped_at | TIMESTAMP | NULL | 停止时间 |

### 3.5 orders（交易订单表）

```sql
CREATE TABLE orders (
    id              BIGSERIAL PRIMARY KEY,
    user_id         INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    api_key_id      INT NOT NULL REFERENCES api_keys(id) ON DELETE CASCADE,
    robot_id        INT REFERENCES robot_configs(id) ON DELETE SET NULL,
    symbol          VARCHAR(20) NOT NULL,
    binance_order_id VARCHAR(100),
    client_order_id VARCHAR(100),
    side            VARCHAR(10) NOT NULL,
    type            VARCHAR(20) NOT NULL,
    price           DECIMAL(36,18),
    quantity        DECIMAL(36,18) NOT NULL,
    executed_qty    DECIMAL(36,18) DEFAULT 0,
    status          VARCHAR(20) NOT NULL DEFAULT 'NEW',
    source          VARCHAR(50) NOT NULL DEFAULT 'manual',
    created_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_api_key_id ON orders(api_key_id);
CREATE INDEX idx_orders_robot_id ON orders(robot_id);
CREATE INDEX idx_orders_symbol ON orders(symbol);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_orders_source ON orders(source);
CREATE INDEX idx_orders_created_at ON orders(created_at);
CREATE INDEX idx_orders_binance_order_id ON orders(binance_order_id);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 自增主键 |
| user_id | INT | FK → users.id |
| api_key_id | INT | FK → api_keys.id |
| robot_id | INT | FK → robot_configs.id（手动下单为NULL） |
| symbol | VARCHAR(20) | 交易对 |
| binance_order_id | VARCHAR(100) | 币安订单ID |
| client_order_id | VARCHAR(100) | 自定义订单ID |
| side | VARCHAR(10) | BUY / SELL |
| type | VARCHAR(20) | LIMIT / MARKET / STOP / TAKE_PROFIT |
| price | DECIMAL(36,18) | 订单价格 |
| quantity | DECIMAL(36,18) | 订单数量 |
| executed_qty | DECIMAL(36,18) | 已成交数量 |
| status | VARCHAR(20) | NEW / PARTIALLY_FILLED / FILLED / CANCELLED / REJECTED |
| source | VARCHAR(50) | manual / robot_{id} |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 四、Redis 缓存设计

| Key 前缀 | 数据类型 | 说明 | TTL |
|----------|---------|------|-----|
| `depth:{symbol}` | Hash | 实时盘口深度 | 1s |
| `klines:{symbol}:{interval}` | List | K线数据 | 5s |
| `price:{symbol}` | String | 最新价格 | 1s |
| `account:{api_key_id}` | Hash | 账户资产缓存 | 30s |
| `robot_status:{user_id}` | Hash | 用户机器人状态 | 10s |
| `rate_limit:{api_key}` | String | 币安API限频计数 | 60s |
| `session:{token}` | Hash | 用户会话 | 24h |
| `otp:{account}` | String | 验证码 | 10m |

## 六、安全设计

1. **api_secret 加密存储**：使用 AES-256-GCM 加密，密钥存储在环境变量
2. **敏感字段脱敏**：查询时 api_key 字段脱敏显示
3. **审计日志**：所有表操作记录到审计日志（使用 PostgreSQL 触发器）
4. **数据备份**：每日全量备份，每小时增量备份（WAL）
5. **连接加密**：PostgreSQL 使用 SSL 连接
