# 接口设计文档

> 系统：CiaoAI 币安合约量化系统（Binance-Futures Trading System）
> 基础路径：`https://api.ciaolab.io/api`
> 协议：RESTful API + WebSocket
> 认证：JWT Bearer Token

---

## 一、通用规范

### 1.1 请求格式

- Content-Type: `application/json`
- Authorization: `Bearer {token}`
- 所有请求通过 HTTPS

### 1.2 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1690123456789
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 200成功，其他为错误码 |
| message | string | 响应信息 |
| data | object/array | 响应数据 |
| timestamp | long | 响应时间戳 |

### 1.3 错误码

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权/Token过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |
| 1001 | 账户或密码错误 |
| 1002 | 用户协议未同意 |
| 1003 | 验证码错误 |
| 2001 | API Key无效 |
| 2002 | 币安API连接失败 |
| 3001 | 机器人参数错误 |
| 3002 | 机器人启动失败 |
| 3003 | 机器人已运行 |
| 4001 | 交易对不存在 |
| 4002 | 余额不足 |
| 4003 | 订单参数错误 |
| 5001 | 风控阈值触发 |

## 二、认证接口（Auth）

### 2.1 登录

```
POST /api/auth/login
```

**请求参数**：

```json
{
  "account": "ciaoaifuture@ciaocex.com",
  "password": "ciao666888ai",
  "agree": true
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| account | string | 是 | 邮箱账号 |
| password | string | 是 | 密码（明文，HTTPS传输） |
| agree | boolean | 是 | 是否同意用户协议 |

**响应**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 86400,
    "user": {
      "id": 1,
      "account": "ciaoaifuture@ciaocex.com",
      "name": "管理员",
      "role": "admin"
    }
  }
}
```

### 2.2 刷新 Token

```
POST /api/auth/refresh
```

**请求参数**：

```json
{
  "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
}
```

**响应**：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 86400
  }
}
```

### 2.3 发送验证码

```
POST /api/notify/sendCode
```

**请求参数**：

```json
{
  "account": "ciaoaifuture@ciaocex.com",
  "type": "view_apikey"
}
```

**响应**：

```json
{
  "code": 200,
  "message": "验证码已发送",
  "data": null
}
```

### 2.4 验证验证码

```
POST /api/notify/verify
```

**请求参数**：

```json
{
  "account": "ciaoaifuture@ciaocex.com",
  "code": "12345678",
  "type": "view_apikey"
}
```

**响应**：

```json
{
  "code": 200,
  "message": "验证成功",
  "data": {
    "valid": true
  }
}
```

## 十、币安 API 代理接口

所有币安原生 API 通过后端代理，前端不直接访问币安。

### 10.1 代理规则

| 前端请求 | 后端转发到币安 |
|----------|---------------|
| `/api/openapi/depth` | `GET /fapi/v1/depth` |
| `/api/openapi/klines` | `GET /fapi/v1/klines` |
| `/api/openapi/trades` | `GET /fapi/v1/trades` |
| `/api/openapi/account` | `GET /fapi/v2/account` + 签名 |
| `/api/openapi/openOrders` | `GET /fapi/v1/openOrders` + 签名 |
| `/api/openapi/historyOrders` | `GET /fapi/v1/allOrders` + 签名 |
| `/api/openapi/getAllPositions` | `GET /fapi/v2/account` + 解析持仓 |

### 10.2 签名逻辑

1. 后端从数据库获取 `api_secret`
2. 将请求参数按字典序排序，拼接为 query string
3. 使用 HMAC-SHA256 签名：`signature = HMAC-SHA256(api_secret, query_string)`
4. 在请求头添加 `X-MBX-APIKEY: {api_key}`
5. 将 `signature` 附加到 query string 中发送
