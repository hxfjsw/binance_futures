# 界面需求：控制面板（Dashboard）

## 页面路径
`/dashboard`

## 功能描述
用户登录后的默认首页，展示多账户资产总览、余额状态、以及所有机器人（常规/AI/风控）的运行状态监控。是全局状态的"一目了然"视图。

## 交互元素清单

### 多账户矩阵表格

| 列 | 说明 |
|------|------|
| 标签 | 账户标签（账户1, 账户2...） |
| apiKey | 脱敏显示（前4位+***+后4位） |
| 域名 | 交易所域名（如 https://****b.io） |
| 账户总净值 | 账户总资产净值（USDT） |
| 可用余额 | 可用 USDT |
| 未实现盈亏 | 持仓未实现盈亏 |

- 表格支持横向滚动（7+ 账户）
- 每行点击可切换到该账户的详情

### 余额卡片

| 元素 | 说明 |
|------|------|
| 可用 USDT | 显示当前可用余额 |
| 冻结 USDT | 显示被冻结的余额 |

### 机器人状态卡片

**卡片结构**：
```
┌────────────────────────────┐
│ 机器人名称          [配置参数]│
│ 描述为空                    │
│ ─────────────────────────── │
│ 未运行              [开关]  │
└────────────────────────────┘
```

**卡片状态**：
- 未运行：灰色卡片，灰色开关
- 运行中：蓝色高亮卡片，蓝色开关，显示运行日期（如 2026/6/15）
- 运行中且有配置：显示"查看"按钮（可查看运行参数）

**卡片分类**：
- 常规机器人（8个）：最低维持刷量、随机刷量、自然刷量、区间刷量、中间价刷量、盘口闪烁、近端深度、分布深度
- AI机器人（3个）：AMM智能做市、推土机模式、智能出货
- 风控机器人（5个）：余额监控、价格监控、余额定时报告、TG日常提醒、TG报警

## 状态与交互

- **机器人开关**：Toggle 切换，切换时发送 `/api/robot/start` 或 `/api/robot/stop`
- **配置参数按钮**：点击弹出 `RobotConfigModal` 弹窗
- **查看按钮**（运行中的机器人）：点击查看当前运行参数，只读模式
- **账户矩阵**：点击某行切换当前操作账户

## 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/openapi/allAccounts` | GET | 获取所有账户列表 |
| `/api/openapi/account` | GET | 获取单个账户资产 |
| `/api/openapi/getIsRunningRobots` | GET | 获取运行中的机器人 |

## 设计要点

- 页面背景：深色主题（#0d0d1a 或类似）
- 卡片：圆角（8px），深色卡片背景（#1e1e2e），带微弱阴影
- 运行中卡片：蓝色边框高亮（#4c6ef5）
- 分类标题：带左侧蓝色竖条（4px宽）
- 表格：表头灰色，行 hover 效果
