# 界面需求：机器人部署（Robot Deployment）

## 页面路径
`/robot-deployment`

## 功能描述
展示所有 19 种量化机器人的"策略商店"界面。用户在此页面浏览所有可用策略，对每个机器人进行配置参数、启动/停止操作。是系统的核心策略管理页面。

## 交互元素

### 机器人卡片

```
┌────────────────────────────┐
│ 机器人名称          [操作按钮]│
│ 描述为空                    │
│ ─────────────────────────── │
│ 状态                [开关]  │
└────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 机器人名称 | 标题（白色粗体） |
| 操作按钮 | 未运行："配置参数"；运行中："查看" |
| 描述 | 默认为"描述为空"（可编辑） |
| 状态 | "未运行" 或 "运行中" + 日期 |
| 开关 | Toggle Switch，点击启停机器人 |

### 状态颜色
- 未运行：灰色卡片（#1e1e2e），灰色开关
- 运行中：蓝色高亮卡片（#4c6ef5），蓝色开关

## 交互行为

- **配置参数按钮**：点击弹出 `RobotConfigModal`（可编辑参数，保存后启动）
- **查看按钮**（运行中）：点击弹出 `RobotConfigModal`（只读模式，查看当前参数）
- **开关**：切换时触发 `POST /api/robot/start` 或 `POST /api/robot/stop`
- **卡片 hover**：轻微上浮效果（translateY(-2px) + 阴影增强）

## 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/robot/list` | GET | 获取所有机器人配置列表 |
| `/api/robot/start` | POST | 启动机器人 |
| `/api/robot/stop` | POST | 停止机器人 |
| `/api/robot/config` | POST | 保存机器人配置 |

## 设计要点

- 卡片布局：3列网格（响应式：大屏3列，中屏2列，小屏1列）
- 卡片间距：16px
- 分类标题：蓝色竖条（4px）+ 白色粗体文字
- 滚动：页面内滚动，侧边栏固定
- 底部：左侧边栏底部有"导入API Key"按钮（固定底部）
